What you need to do in Firestore

So that each email maps to one outlet and different emails can see different data:
Create an outlet per branch (if needed)
In outlets, e.g. outlet_001, outlet_002, with at least name (and any other fields you use).
Create one user document per Firebase Auth user
In the users collection:
Document ID = that user’s Firebase Auth UID (from Authentication → Users in Firebase Console).
Add field outletId (string) = the outlet id for that branch (e.g. outlet_001 or outlet_002).
Optionally: role, displayName.
Example:
User A (email manager@spa.com) → create users/<uid_of_A> with outletId: "outlet_001".
User B (email branch2@spa.com) → create users/<uid_of_B> with outletId: "outlet_002".
Then:
Logging in as A loads only outlet_001 data.
Logging in as B loads only outlet_002 data.
If both users currently see the same data, their users/{uid} documents probably have the same outletId. Change the second user’s document to a different outletId (and ensure that outlet exists in outlets). No code changes are required beyond what’s above; the app now uses only the outletId from users/{uid} and never a default outlet.




In Firebase Console go to Firestore Database.
Open the users collection.
If there is no document whose Document ID is that UID:
Click Add document.
Set Document ID = the UID you copied (paste it exactly).
Add a field: name outletId, type string, value e.g. outlet_001 (or whatever outlet this user should use).
Optionally add: role (string, e.g. "staff" or "admin"), displayName (string).
Save.
If a document with that ID already exists but the user still gets “no outlet assigned”:
Open that document.
Add or fix the field outletId (string), e.g. outlet_001, and save.
